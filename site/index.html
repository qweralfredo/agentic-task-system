Open Source · MCP-Native · Agentic-First · 100% Local

Orchestrate a cluster of AI agents inside a single project

Pandora Todo List is an open-source Agentic Task System built on the Model Context Protocol (MCP). It gives every AI coding agent — GitHub Copilot, Claude, Cursor, and others — a shared, persistent operational context: backlog, sprints, work items, wiki, and knowledge checkpoints.

Everything runs 100% locally on your machine — no cloud account, no SaaS, no data leaving your environment. One docker compose up -d and the full stack is live: API, frontend, MCP server, and PostgreSQL.

The Problem

Why a dedicated agentic system?

When you use multiple AI coding agents on the same project — say, GitHub Copilot for implementation, Claude for architecture reviews, and a custom agent for testing — each one starts without context from the others. They re-discover the same decisions, repeat the same analysis, and can even contradict each other.

Pandora solves this with a shared operational memory: a persistent, structured store that all agents can read from and write to through a standard protocol. Every sprint decision, every backlog priority, every architecture risk gets recorded and becomes immediately available to any agent that joins the project.

The result is a cluster of agents that behaves like a coordinated team — each one picks up where the last left off, speaks the same task language, and never wastes tokens re-establishing context.

✗ Without Pandora
  • Each agent starts cold
  • Context lives only in chat history
  • Decisions are re-made every session
  • Agents duplicate or contradict work
  • No audit trail of agentic actions
  • Human must bridge every context gap
✓ With Pandora
  • Every agent shares the same backlog
  • Context persisted in PostgreSQL
  • Checkpoints capture decisions & risks
  • Agents hand off work via work items
  • Token-tracked run log per agent
  • Humans monitor, agents execute
Architecture

How the agentic cluster works

01

Agent connects via MCP

Any agent IDE (VS Code Copilot, Cursor, Claude Desktop…) adds Pandora's MCP server as a tool provider. The server runs in Docker at http://127.0.0.1:8481/mcp and exposes all operations as typed MCP tools.

↓
02

Agent reads project context

Before coding, agents call project_list, backlog_list, and read pandora://projects/{id}/context — an MCP resource that returns the full sprint state, open work items, wiki, and last checkpoint in a single call.

↓
03

Agent works tasks, updates status

While implementing, agents call workitem_update to mark items InProgress → Done, logging the model name, IDE, tokens used, and a feedback summary. The next agent can instantly see what was done and why.

↓
04

Agent saves knowledge checkpoints

At key milestones agents call knowledge_checkpoint with a context snapshot, key decisions, identified risks, and suggested next actions. Future agents (or the same one on a new session) load these checkpoints to resume without context loss.

↓
05

Human reviews on the dashboard

The React frontend shows the sprint board, agent run history, wiki pages, and all checkpoints in real time. Humans can adjust priorities, reject work items, or add new backlog items — and agents pick those changes up on the next session.

MCP Tools

What agents can do

Every operation exposed via MCP is a first-class tool that any compliant agent can call. Here are the key tools Pandora exposes:

📂 Projects & Backlog
project_list List all projects
project_create Create a new project
backlog_add Add a backlog item with priority & story points
backlog_list List all backlog items
🏃 Sprints & Work Items
sprint_create Create a sprint from selected backlog items
workitem_list List work items with statuses
workitem_update Update status, log model, IDE & tokens used
🧠 Knowledge & Context
knowledge_checkpoint Save context snapshot, decisions, risks
documentation_add Add a documentation page
wiki_add Add a wiki page to the project
📊 MCP Resources
pandora://projects/{id}/context Full project state in one call
pandora://projects/{id}/backlog Current backlog
pandora://projects/{id}/knowledge Wiki + checkpoints
pandora://projects/{id}/tasks/triage Prioritized task triage view
